# LifeMirror 完整设计方案

---

### 一、项目背景与目标

**LifeMirror** 是一个构建"数字人生孪生"的个人历史记录系统。它旨在将一个人一生的文字、影像、声音、社会关系等多元信息，通过结构化存储与智能分析，编织成一部可查阅、可交互、可传承的"个人历史"。系统不仅要记录客观事件，还要捕捉情感、思想和关系演变，最终形成数字化的"人生双胞胎"。同时，系统必须融入中国家族文化，支持家谱数字化、祖先关联、五服计算等特色功能。

**核心目标**：
- **隐私第一**：所有人生数据本地加密存储，用户完全掌控。
- **离线优先**：无需依赖云端，随时随地记录和查阅。
- **家族传承**：构建跨越代际的数字家族树，支持复杂宗族关系。
- **智能加持**：利用 AI 自动标注、分析情感、生成叙事。
- **跨平台一致体验**：覆盖移动端、桌面端、Web 端。

---

### 二、总体架构设计

系统采用分层架构，以 **Rust 核心库** 为业务逻辑中枢，通过 FFI 供各前端调用。数据存储层包括加密的 SQLite（结构化数据）、MinIO（多媒体文件）和嵌入式 Neo4j（关系图谱）。智能分析模块以 ONNX Runtime 为载体，在本地运行 AI 模型。

```

│                     前端层 (Flutter)                           │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐    │
│  │ 时间线界面  │ │ 家族树可视化 │ │ 多媒体画廊/VR空间   │    │
│  └─────────────┘ └─────────────┘ └─────────────────────┘    │
│                           ↓ (dart:ffi)                        │
├─────────────────────────────────────────────────────────────┤
│                  Rust 核心库 (lifemirror_core)                 │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ 数据模型 (事件/人物/关系) │ 加密引擎 (基于Tink)       │   │
│  ├──────────────────────────────────────────────────────┤   │
│  │ 图谱引擎 (neo4j 客户端封装)  │ 多媒体索引 (MinIO客户端)│   │

│  └─────────────┘ └─────────────┘ └─────────────────────┘    │
```

### 三、模块详细设计

#### 3.1 安全存储层

**技术选型**：
- **结构化数据加密**：SQLCipher（SQLite 加密扩展），256-bit AES 加密。
- **密钥保护**：Android Keystore / iOS Keychain / Windows TPM / Linux 秘密服务。
- **文件加密**：Tink 的 AEAD 接口，分块加密大文件。
- **跨平台封装**：Rust 调用各平台安全 API（通过 `android-keystore-rs`、`keychain` 等 crate）。

**核心逻辑**：
- 用户首次启动时生成主密钥，由平台 Keystore 保护。主密钥永不离开硬件安全模块。
- 所有写入 SQLite 的数据自动加密（SQLCipher 透明加密）。
- 照片、视频等大文件上传时通过 Tink 流式加密后存入 MinIO，每个文件使用独立随机密钥，该密钥由主密钥派生并加密存储于 SQLCipher 中。
- 敏感操作（如查看私密日记）可配置需要生物认证（指纹/面容），由平台生物识别 API 配合密钥使用策略实现。

#### 3.2 图谱引擎

**技术选型**：
- **数据库**：桌面端使用 Neo4j 社区版（嵌入或本地服务），移动端使用 SQLite 存储简化关系，通过定期同步或在桌面端进行复杂分析。
- **Rust 客户端**：`neo4rs` 通过 Bolt 协议连接 Neo4j；移动端使用 `sqlx` 操作 SQLite。

**功能模块**：
- **人物节点**：存储姓名、生卒、字辈、头像指针、生平简介等。
- **关系边**：支持父子、夫妻、兄弟、堂表、过继、入赘、兼祧等关系类型，每条边可包含有效时间（如过继关系的时间范围）。
- **查询能力**：
  - 亲属关系路径查找（如计算两人之间的亲属关系）。
  - 五服计算：根据传统礼制计算某人去世时五服内亲属。
  - 家族树展开：按辈分获取特定支系的所有人物。
  - 祖先追溯：从当前人物向上追溯所有直系祖先。

**存储策略**：
- 桌面端：用户安装时可选择安装 Neo4j 社区版（提供安装包或 Docker 镜像），或使用内置的轻度图数据库（如 `sled` + 关系索引）。
- 移动端：以 SQLite 存储关系表，支持基本的查询（如获取父母、子女），复杂分析通过导出数据到桌面端或云端进行。

#### 3.3 多媒体管理

**技术选型**：
- **对象存储**：MinIO（本地部署，支持 S3 API），可部署在用户本地网络或同一设备上，确保数据不离开用户控制范围。
- **媒体处理**：FFmpeg（通过 Rust 的 `ffmpeg-next` 绑定）进行缩略图生成、格式转码、元数据提取。
- **元数据提取**：使用 `exif` 库读取照片 EXIF 信息，提取拍摄时间、地点、设备等。

**工作流程**：
1. 用户导入照片/视频 → 后台任务提取 EXIF 时间、地点，生成缩略图和预览图。
2. 调用人脸识别模型检测照片中的人脸，生成人脸特征向量，并与家族树中的人物进行匹配（自动或人工确认）。
3. 原始文件通过 Tink 流式加密后存入 MinIO，元数据（包括时间、地点、人脸列表、缩略图路径等）存入 SQLCipher。
4. 视频文件自动转码为 H.264 通用格式（保留原始文件），并生成字幕文件（若语音识别可用）。

#### 3.4 智能分析层

**技术选型**：
- **推理框架**：ONNX Runtime（支持 CPU/GPU，提供 Rust API）。
- **模型**：
  - 人脸识别：InsightFace（转换为 ONNX），用于检测和识别照片中人物。
  - 语音转文字：Whisper.cpp（轻量，可封装为 ONNX 或直接集成）。
  - 情感分析：基于文本的 DistilBERT 小型模型（ONNX），分析日记、信件等文本情感。
  - 场景识别：MobileNet（ONNX），自动为照片添加场景标签（如"海滩"、"聚会"）。
- **服务模式**：AI 模块作为独立线程运行，通过消息队列（如 `crossbeam_channel`）与核心库通信，避免阻塞 UI。

**功能列表**：
- **自动人物标注**：导入照片时自动识别其中人物，并建议与家族树中人物关联。
- **视频语音转录**：将视频中的语音转为文字，生成时间轴字幕，并建立全文索引。
- **文本情感分析**：分析日记、信件等文本内容，生成情绪曲线（随时间变化的情绪波动）。
- **人生统计报告**：基于事件和多媒体数据，自动生成统计图表（如"过去十年去过的城市"、"社交活跃度变化"）。
- **关系亲密度推测**（未来）：通过合影频率、共同事件等，推测人物之间的关系亲密度。

#### 3.5 时间锁机制

**设计目的**：实现"死后公开"、"指定日期解锁"等功能，确保敏感资料在特定时间点或条件下才能被访问。

**技术实现**：
- 基于 **Shamir 秘密共享** 将加密密钥拆分为多份，分别存储于用户指定的可信实体（如家人设备、公证服务、时间锁服务）。
- 结合 **RFC 3161 可信时间戳** 证明解锁条件的真实性（防止本地时间篡改）。
- 本地简化版：将密钥与系统时间绑定，到期自动释放（适合普通用户，但存在时间篡改风险）。

**流程**：
1. 用户选择资料并设定解锁日期（如 2050-01-01）或条件（如"我去世后"）。
2. 系统生成随机密钥，用该密钥加密资料。
3. 将密钥拆分为 3 份，分别交给 3 位信任的联系人（加密存储在他们的设备），或上传到可信时间戳服务。
4. 到期后，至少 2 位联系人同意并发送密钥份额，系统恢复密钥并解密资料。
5. 若条件为"我去世后"，需结合数字遗产继承流程（如定期心跳检测、多方证明）。

#### 3.6 前端 UI

**技术选型**：Flutter 3.x（支持移动、桌面、Web）。

**关键页面**：
- **时间线**：按年/月/日展示事件，支持过滤（按类别、人物、地点）和搜索。点击事件可查看详情及关联多媒体。
- **家族树**：基于关系图谱数据，绘制可交互的家谱图（支持缩放、展开/折叠、高亮某支系）。点击人物节点可查看生平简介及关联事件。
- **多媒体画廊**：按时间、人物、地点分组展示照片视频，支持人脸聚类浏览。可播放视频、查看照片 EXIF 信息。
- **人生地图**：在地图上标记迁徙轨迹和生活过的地点，点击地点可查看相关事件和多媒体。
- **智能报告**：展示情绪曲线、社交活跃度、健康趋势等统计图表，支持时间范围选择。
- **纪念空间**：可为已故亲人创建公开或私密的纪念页面，支持留言、献花、点烛等功能（需权限控制）。

**与 Rust 核心交互**：通过 `dart:ffi` 调用 Rust 编译的动态库，传递 Protocol Buffers 序列化的数据。所有耗时操作（如 AI 分析、加密）在 Rust 侧异步执行，避免阻塞 UI。

### 四、数据模型（核心实体）

#### 4.1 事件（LifeEvent）
- `id`：UUID
- `title`：标题
- `description`：描述
- `timestamp`：绝对时间（Unix 毫秒）
- `relative_time`：相对时间描述（如"时年7岁"）
- `location`：GeoLocation（经纬度+地名）
- `participant_ids`：参与人物 ID 列表
- `evidences`：证据文件 ID 列表
- `categories`：分类标签（如"教育"、"工作"、"家庭"）
- `emotion`：情感标签（喜悦/悲伤/平静等，可由 AI 分析或手动标注）

#### 4.2 人物（Person）
- `id`：UUID
- `name`：姓名
- `birth_name`：本名/字辈名
- `gender`：性别
- `birth_date`：出生日期
- `death_date`：逝世日期
- `biography`：生平简介
- `profile_photo`：头像文件 ID
- `family_relations`：家族关系列表（目标人物 ID、关系类型、开始/结束时间）
- `social_relations`：社会关系列表（目标人物 ID、关系类型、相识时间）
- `generation`：辈分（相对于家族始祖）

#### 4.3 关系类型（RelationType）
- **血缘**：`Father`, `Mother`, `Son`, `Daughter`, `Brother`, `Sister`, `Grandfather`, `Grandmother`, `Cousin`, `Uncle`, `Aunt` 等。
- **姻亲**：`Husband`, `Wife`, `FatherInLaw`, `MotherInLaw` 等。
- **宗族特殊**：`AdoptedSon`, `AdoptedDaughter`, `SonInLawUxoril`（入赘）, `JointHeir`（兼祧）等。

#### 4.4 证据文件（Evidence）
- `id`：UUID
- `type`：图片/视频/音频/文档
- `file_path`：MinIO 中的存储路径（加密后）
- `thumbnail_path`：缩略图路径（加密后）
- `mime_type`
- `size`
- `created_at`：文件创建时间（从 EXIF 或用户指定）
- `encryption_key_id`：加密密钥标识（用于查找对应的加密密钥）

#### 4.5 家族树元数据（FamilyTree）
- `id`：UUID
- `name`：家族堂号
- `ancestor_id`：始祖人物 ID
- `genealogy_text`：原始族谱文字（OCR 结果）
- `migration_history`：迁徙记录列表（地点、时间、原因）

### 七、关键挑战与对策

| 挑战 | 对策 |
|------|------|
| **移动端 Neo4j 嵌入** | 移动端使用 SQLite 存储简化关系，定期同步至桌面端；或使用 Neo4j 的轻量级移动库（尚未成熟） |
| **多媒体文件加密性能** | 使用 Tink 流式加密，分块读写；异步处理，避免阻塞 UI |
| **模型在移动端的运行效率** | 选择轻量模型（如 Whisper.cpp 已优化）；使用 ONNX Runtime 的 CPU/GPU 加速；必要时降级到云端（用户选择） |
| **时间锁的可信实现** | 初期使用本地时间+加密信封（可被绕过），后期引入可信第三方（如家人设备、公证服务） |
| **家族树可视化复杂度** | 使用 Canvas 自定义绘制，分层渲染；支持缩放和节点聚合 |
| **数据安全与备份** | 提供加密导出/导入功能；支持用户自行备份到云存储（加密后） |

### 八、总结

本方案基于 **Rust + Tink + Neo4j + Flutter** 技术栈，完全符合 LifeMirror 的核心需求：
- **隐私**：硬件级加密 + 生物认证 + 时间锁，用户完全掌控数据。
- **离线优先**：所有组件本地运行，无需网络依赖。
- **家族传承**：图数据库原生支持复杂关系，兼容中国宗族特色。
- **智能**：ONNX Runtime 本地运行 AI 模型，保障隐私的同时提供智能分析。
- **跨平台**：Flutter 覆盖移动、桌面、Web，Rust 核心库一次编写多处运行。

该方案充分考虑了项目的长期发展和可维护性，所有技术选型均为领域内成熟方案，且具备良好的社区支持和生态。团队可依据实施路线图，分阶段迭代开发，逐步交付价值。
